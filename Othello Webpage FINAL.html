Othello

by: Sophie Zeng

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

As a child, my family and I played Othello a lot, and I thought that coding Othello would be fun and make it more convenient for us to play. I found Othello to be an interesting and unique game; one that probably not many have coded before.

2. What I Learned

Through user testing, I found quite a lot of errors and mistakes. First, while having my mom test the game, I found that I did not code anything in the scenario that one player does not have any moves left. So, I used booleans to check whether there were any valid plays left. When I tested with the class, I realized that I had not put instructions, so I added a pop up with the basic rules of Othello. I also learned that while coding a game, you really have to switch to a coding mindset, and think in terms of every layer of code you have to write rather than just the game rules.

3. Architecture of My Game

I have an Othello class, a GamePanel class, a GameBoard class, and a Square class. In the Othello class, I just make the basic frame of the game, making the JFrame and GamePanel. I make the actual board in the GameBoard class, and I set the size of the board and each square. In the Square class, I make the basic pieces, and if the status of the piece is set to white, then the piece will actually turn white. The GamePanel class is where I code how the board and different states (menu, game, and end) will look, and set up the real working parts of the game. There, I manipulate most of what happens on the board, and do all the checking.

4. Technical Implementation

While coding the checking, I used multiple nested if statements and for loops to thoroughly check every layer and tell the program what to do in each scenario that could happen. First, I checked whether the space clicked was empty or not. Then, I checked for a tail of the sandwich, and made sure the checking stopped at the tail. I then went through the pieces inside the sandwich, checking that none of them were empty or of the same color as the current turn. If the play made it through all of those checks, then it was added to a list of points that would be flipped (changed color). The rest of the game was somewhat simple, going off of the frame of LeagueInvaders.

5. Most Challenging Part

Coding the checking was by far the most difficult and complicated part, but also the code that I am most proud of. Othello took me quite a while to code, longer than I'd like to admit. I coded the checking 4 times, the first 3 times all having errors that I did not know how to fix. After lots of frustration and random errors popping up, I completely restarted the checking part of the code, and changed my logic about the game rules. I went layer by layer, slowly checking every aspect of what makes a valid play. I did find more errors along the way, but I fixed them, and I am quite proud of the final product.

6. Future Fixes and Updates

If I had unlimited time to keep working on this project, I would make the code a neater and more precise. My code is a bit messy, and quite complicated for anyone to read besides me, so I would organize my code better. Adding sound effects would also enhance the user experience, but I am still very satisfied with my final product.
